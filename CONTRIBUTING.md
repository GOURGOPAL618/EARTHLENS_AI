# Contributing to EarthLens AI

Thank you for your interest in contributing. All contributions are welcome —
bug fixes, new analysis modules, documentation improvements, or ideas.

## How to Contribute

1. Fork the repository
2. Create a feature branch
```bash
   git checkout -b feature/your-feature-name
```
3. Make your changes
4. Test thoroughly
5. Commit with a clear message
```bash
   git commit -m "feat: add flood detection module"
```
6. Push and open a Pull Request

## Commit Message Convention

| Prefix | Use for |
|---|---|
| `feat:` | New feature or module |
| `fix:` | Bug fix |
| `docs:` | Documentation only |
| `refactor:` | Code restructure, no feature change |
| `data:` | Dataset or pipeline changes |
| `model:` | ML model changes |

## Code Style

- Follow PEP 8
- Add docstrings to all functions
- Keep modules independent — no cross-imports between analysis engines

## Reporting Bugs

Open an issue with:
- What you did
- What you expected
- What actually happened
- Python version and OS
